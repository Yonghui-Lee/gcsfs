
## 2024-05-18 - Fast String Parsing for Core IO Paths
**Learning:** In highly trafficked file system logic (`_split_path`, `_strip_protocol`), importing and instantiating parsing components like `urllib.parse.urlsplit` and `parse_qs` creates considerable object allocation overhead. In `gcsfs/core.py`, manually parsing URL parameters (`find("#")`, `find("?")`, `split("&")`) yielded a 2x speedup compared to standard library functions. Similarly, for generic `_strip_protocol` path operations, avoiding fallback conversions from path objects to URI mappings and relying on direct `string.find(":")` matching when stripping `protocol://` saves overhead without breaking compatibility with native path interfaces (e.g. `stringify_path`).
**Action:** When a method is called frequently (especially in critical/hot path operations like list iterations or path resolution), try to rely on native string slicing, direct substring indexing, and simple loops over creating temporary objects (like lists or sets in `_coalesce_generation`).
